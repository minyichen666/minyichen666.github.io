Greedy Algorithm

Greedy algorithm solves problem by making locally best choices which collectively contribute to optimal(or suboptimal) solution.

There is a greedies in this algorithm, i.e. it will

leetcode problem (my solution)

445. Assign Cookies

Locally best choice: satisfy each child with smallest possible cookies

376. Wiggle Subsequence

Locally best choice: increase length of wiggle sequence if at peak/valley; otherwise, treat new value as current valley/peak.

53. Maximum Subarray

Locally best choice: add value to subarray sum if sum is positve; discard subarray if sum is positive.

122. Best Time to Buy and Sell Stock II

Locally best choice: buy stock at previous day and sell stock at current day if its price is higher than its price at previous day; otherwise, don’t buy at previous day.

45. Jump Game II

Locally best choice: jump as far as possible with ith jump until reach to the destination

1005. Maximize Sum Of Array After K Negations

Locally best choice: flip minimum number in the array as long as k > 0, and decrement k by 1 with each flip.

134. Gas Station

Locally best choice: If the accumulate gas become negative(not feasible), start at current station; if the accumulate gas is positive(still feasible), go ahead.

Detailed explaination: Notice that car will not start at station where cost is greater than gas. The best choice is to start at the station where cost is less than gas and right after the station where cost is greater than gas. If accumulate gas become negative, that means that car just passes through the station where cost is greater than gas, and you should choice different starting point.

135. Candy

It is a hard problem. I got really close to the solution, but eventually I have to check the answer.

Locally best choice:

  • Iterate from begin to end: if ratings[i] > ratings[i - 1], candies[i] = ratings[i - 1] + 1. (Greediness: just give the child 1 more candy, if he get higher rating)

  • Iterate from end to begin: ratings[i] > ratings[i + 1], candies[i] = ratings[i + 1] + 1. (Greediness: just give the child 1 more candy, if he get higher rating)

860. Lemonade Change

Locally best choice: greedy! If have 10, use 10 first.

406. Queue Reconstruction by Height

Locally best choice: After sort 2d array, insert the people[i] from large people[i][0] to small people[i][0]. And depending on the value of people[i][1], we insert the people[i] into return queue.

Because we insert the people[i] from large people[i][0] to small people[i][0]. At current iteration,we only need to insert ````people[i]atpeople[i][1]th position of queue, which will make it satisfy the condition. All people[i]inserted in the future iteration is smaller than currentpeople[i], and they will not affect people[i][1]```

968. Binary Tree Cameras

  • Greediness: install minimum number of camera at each subtree

  • Locally best choice:

    1. if any of the child node is not covered, a camera should be installed current node needs

    2. if none of the child node is not covered, and at least one child node has camera, current node is covered

    3. if all of the child node have camera, current child node does not have to install camera with the consideration to minimize cameras
