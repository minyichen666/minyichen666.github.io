Some notes

vo(value object) is used for accepting object from requests, and sent to the client side. We can use it for converting to po

@Transaction Annotation requires @EnableTransactionManagement annotation

BeanUtils.copyProperties(Object source, Object target) is useful in copying the properties from the source that have the same name with the target

It is not recommended to ask service layer to return vo object. Because other controller class may also want to use this service method. Therefore, we should instead return entity object to increase reproducibility.

Controller: Process request, accept, and validate data; Receives data processed by the Service and wraps it into the specified VO (Value Object) for the page.

Service: Receives data from the Controller and handles operations.

We should use BigDecimal instead of float for variable

Use Long type for id field of entity object

It is recommended to use a common service that contains the dependencies or classes for all other services to use, and other services can depends on the common service.

How to use feign to call method from other service?

  1. import open-feign dependency
  2. use @EnableDiscoveryClient for both caller and callee; configure nacos in caller and callee
  3. use @EnableFeignClient annotation
  4. use @FeignClient in the service layer interface

Use to for the object that transmits between services

Troubleshooting

p18

  1. Error: Property ‘ Sqlsessionfactory ‘ or ‘ sqlsessiontemplate ‘ is required

Solution: Downgrade Spring boot from 3 to 2, and Downgrade Spring Cloud to corresponding version, too. This is because Spring boot 3 does not support Mybatis Plus.

p21

  1. On Windows, launching nacos/bin/start.cmd keeps showing Nacos is starting.

Solution: run start.cmd in terminal with startalone mode

  1. java.lang.IllegalArgumentException: The specified key byte array is 0 bits which is not secure enough for any JWT HMAC-SHA algorithm.

Solution: Go to nacos/config/application.properties update “nacos.core.auth.default.token.secret.key” using generated secret key from https://www.sojson.com/base64.html

p22

  1. Error Caused by: java.lang.IllegalStateException: No Feign Client for loadBalancing defined. Did you forget to include spring-cloud-starter-loadbalancer?

Solution: This is because Spring Cloud Feign start using spring-cloud-loadbalancer instead of ribbion. Therefore, add spring-cloud-loadbalancer dependency to your pom file.

62

  1. Access Denied when uploading file to s3 from spring boot project

Solution: set Environment Variables - AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY (RECOMMENDED since they are recognized by all the AWS SDKs and CLI except for .NET), or AWS_ACCESS_KEY and AWS_SECRET_KEY (only recognized by Java SDK)
