秒杀系统

  1. Use CDN
    1. A Content Delivery Network (CDN) is a distributed network of servers strategically located around the world, designed to deliver internet content to users more quickly and efficiently. CDNs are used to serve a wide variety of content types including web pages, images, videos, and downloadable objects (documents, software, media files, etc.).
      The primary goal of a CDN is to improve web performance by reducing the distance between the server and the user, minimizing latency and packet loss.
  2. Set up page caching to return cached pages for the same page and uid over time
  3. Use database cache as much as possible
  4. Use strong consistency on write operation, use weak consistency on read operation
  5. Cache product information and other relevant data that does not change much on each physical machine.
  6. Dynamic data like inventory is cached for a certain amount of time (typically a few seconds) using passive expiration, and then the Tair cache is pulled for the latest data after expiration.
  7. Service Isolation
    1. Elatically increase or decrease system quota for services
    2. Services failure does not affect each other
  8. System isolation
    1. Elatically Add or remove node for services
    2. load balancing
  9. Data Isolation: Isolating data for spike system from main system
    1. Isolating cache
    2. Isolating database

consistency

  1. Strong Consistency:
    guarantees that every read operation returns the latest write operation’s result, regardless of the node on which the read operation is executed.

  2. Weak Consistency:
    updates to a data value are not guaranteed to be immediately visible to all participants or nodes in the system. This means that after a write operation completes, subsequent read operations may not immediately see the updated value.
